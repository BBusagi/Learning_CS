# CAP Theorem
## CAP基本概念
+ C Consistency 一致性
  + 所有节点同一时间看到的资料都是一致的
+ A Availability 可用性
  + 所有请求都可以在有限时间获取回应
+ P Partition Tolerance 分区容错
  + 在网络分区或者通讯丢失的时候，系统仍然可以运转
+ 网路分区 节点之间无法通讯，被分为了孤立子集
+ 抉择 CP牺牲可用性 AP牺牲一致性
+ 通常无法放弃分区容错性

## CP vs AP
+ 通常**默认选择AP**牺牲一致性
+ 只有当场景无法容忍过期数据的时候才会选择牺牲可用性
+ 强一致性场景案例 库存管理，有限资源预定，银行系统
