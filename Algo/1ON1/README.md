# 1ON1 总览

这个目录记录了我与不同高级工程师的 1ON1 对话整理版。目标不是保留逐字速记，而是沉淀可回看、可执行、可持续迭代的职业发展建议。

## 文件说明
- [QunQiao.txt](./QunQiao.txt)：偏职业路径、转型策略、简历表达。
- [Cai.txt](./Cai.txt)：偏 XR / CV / 底层工程能力、soft skills、简历定位。
- [DennyKu.txt](./DennyKu.txt)：偏大公司与小公司差异、求职策略、工程师长期竞争力。
- [QuestionList.txt](./QuestionList.txt)：1ON1 提问模板与长期检查项。

## 共通结论

### 1. 先明确目标，再反推准备路径
- 不要只盯着下一份工作，而要先想清楚 3 到 5 年后希望成为什么类型的工程师。
- 职业选择要同时看兴趣、市场机会和长期可迁移性。

### 2. 简历必须从“描述工作”升级成“证明价值”
- 重点不是列职责，而是说明你解决了什么问题、承担了什么责任、带来了什么结果。
- 不同岗位应准备不同版本的简历，突出对应能力标签。

### 3. senior 往上走，soft skills 不再是附加项
- 需要能够理解完整流程，参与前期需求，做跨团队沟通，并对质量和结果负责。
- 个人成长不能只补技术深度，也要补 ownership、表达和协作能力。

### 4. 求职准备要同时经营“能力”和“曝光”
- 技术准备仍然重要，但拿到面试机会本身也是关键能力。
- 需要同步经营简历、公开项目、GitHub、猎头渠道、社区参与和行业连接。

### 5. 大公司经验的价值主要在规模和复杂度
- 大公司不一定让你做得更杂，但通常会让你面对更大规模的系统、更成熟的协作流程和更高影响范围的问题。
- 如果目标是长期成长，这类经历通常有更强的迁移价值。

## 建议的下一步
1. 明确主线方向，至少在“XR / 图形”“通用 SDE”“平台 / 后端”“AI 相关”中先选一个主方向和一个备选方向。
2. 为目标方向分别整理简历版本，补齐结果导向表达。
3. 用 `QuestionList.txt` 继续迭代后续 1ON1，避免每次都从零准备问题。
4. 每次对话后补充三个字段：结论、行动项、下次追问。
