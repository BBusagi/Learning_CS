# Cai

## 对方背景
- 职涯路径涵盖 HTC、Pretia、MediaTek。
- 经历横跨 CV、PM、Technical Manager 等角色，说明其对技术深度与组织协作两侧都有实务观察。
- 技术经历中包含 CV 专精方向，以及 NDK / SDK 等偏底层工程的实际开发经验。

## 讨论重点
- 如果继续走图形 / CV / XR 相关方向，应如何建立个人竞争力。
- senior 工程师与更上层工程师真正被看重的能力是什么。
- AI、XR、底层工程这些方向近几年的变化和机会点在哪里。

## 核心观点

### 1. 技术深度之外，soft skills 会越来越重要
- 越往上走，评价标准不只是代码能力，还包括专业判断、沟通协作、主动性和领导力。
- 上层工程师的重要职责之一，是对整体质量和流程把关，而不只是完成某一段开发任务。
- 当你能理解完整流程、参与前期企划、提前识别需求和风险时，你的价值会明显提高。

### 2. 个人优势要从“能做”升级为“值得被选”
- 简历和面试都要体现你真正想展示的能力，而不是平均罗列所有经历。
- 面试官通常会综合看年资与成长轨迹、项目内容和技术复杂度、项目周期与个人角色、是否有规模化经验，以及你能否清楚讲出真正投入和真正理解的部分。

### 3. 底层与专精方向仍有价值，但要结合市场表达
- 如果继续往 CV、XR 或底层系统方向发展，需要同时回答两个问题：我在技术上到底擅长什么，以及这些能力如何对应当前市场需求。
- 仅有技术兴趣不够，还要能够把能力映射到产品价值、业务场景和行业趋势上。

### 4. 可以参考的相关发展路径
- 对话中也提到了一些相关岗位样本，例如游戏、Unity 底层、工作室研发等方向。
- 这些路径说明，图形、XR、CV、底层平台能力并不只对应单一路线，而是可以映射到更广的交互、引擎和产品研发场景。

### 5. AI 小团队的典型工作重心
- 对话中提到，AI 小公司常见工作集中在 fine-tune / retrain、模型搭建和数据清洗。
- 这意味着如果想往 AI 靠拢，除了模型概念本身，也要理解数据、训练流程和落地约束。

## 可提炼的职业建议
- 若目标是更高阶工程岗位，不能只强化“开发能力”，还要同步强化“流程视角”和“跨团队协作能力”。
- 简历应该围绕目标岗位做高亮，主动选择最能代表自己优势的内容，而不是追求面面俱到。
- 如果未来想在 XR + AI、CV 或底层平台方向发力，需要尽快形成一套能被外部看懂的个人标签。

## 后续行动
- 梳理自己最强的 2 到 3 个技术标签，例如 XR 工程、图形 / 视觉相关能力、跨端产品落地经验。
- 为关键项目补充“规模、难点、决策、结果”的表达，强化 senior 视角。
- 在准备下一版简历时，加入对市场趋势的映射，例如 XR、AI、3D 交互、跨平台底层能力等。
- 有意识地补齐 soft skills 证据，例如跨团队推动、需求澄清、质量把关、项目 ownership。
