# 算法时空复杂度分析实用指南
### 借助复杂度反推解题思路
+ 注意题目中给出的数据规模，通过算法复杂度反推算法类别
+ 一般来说10⁸是时间复杂度上限，超过即会TLE；空间复杂度的上限为10⁷
+ 如果数组长度可达到10^6,那这题就不可能写成On^2，要尽量做到𝑂nlogn或On
+ 如果数据N的长度为20这种小规模，即允许阶乘或者指数级运算
+ **需要考虑完题目的所有信息，再去考虑代码**

### 由于编码失误导致复杂度异常
+ 使用了标准输出
+ 错误地「传值」而非「传引用」
+ 接口对象的底层实现不明

### Big O 表示法
+ 符号O代表一个函数的集合
+ 只保留增长速率最快的项，其他的项可以省略
+ Big O 记号表示复杂度的「上界」（最坏情况下增长速度的上限）

### 非递归算法分析
+ 嵌套循环的复杂度就是每层相乘
+ 还需要考虑每层具体在干什么
+ 例如 滑动窗口，双循环但是指针不回退，所以不是On^2而是On

### 数据结构分析
+ 当衡量数据结构类中的某个方法的时间复杂度，不能简单地看最坏时间复杂度，而应该看摊还（平均）时间复杂度
+ 计算平均时间复杂度最常用的方法叫做「聚合分析」

### 递归算法分析
+ 递归算法就是遍历一颗递归树
  + 递归算法的时间复杂度 = 递归树的节点个数 x 每个节点的时间复杂度
  + 递归算法的空间复杂度 = 递归树的高度 + 算法申请的存储空间
